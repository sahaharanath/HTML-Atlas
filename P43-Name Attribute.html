(1) Form without Using Name Attribute:

 

  After submission, the URL appears as: http://localhost:63342/action?

  URL : .../action?


  If we want to pass data through the URL after form submission, the 'name' attribute comes in handy.

  (2) Form Using Name Attribute:

 

  After submission, the URL appears as: http://localhost:63342/action?User-ID=XYZ&Login-Password=12345

  URL : ...action?User-ID=XYZ&Login-Password=12345

  By entering User ID: XYZ and Password: 12345, the submitted data is visible in the URL.

  This shows that data is appended to the URL after the given action (e.g., '/action' followed by '?').

  Note: This method is not secure for sensitive information, and safer methods will be explored later.
